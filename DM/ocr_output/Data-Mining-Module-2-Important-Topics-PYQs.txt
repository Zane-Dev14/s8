5. Explain the two sampling methods used in data reduction.

===== PAGE 10 =====
Sampling is a technique in data reduction where we select a small representative subset of a
large dataset, so that the analysis becomes faster and easier without much loss of information.
important sampling methods are:
1. Simple Random Sample Without Replacement (SRSWOR)
e We randomly select s tuples (records) from the dataset D, where s < N (N is the total
number of tuples).
e Important Point: Once a tuple is selected, it is not put back into the dataset.
e So, each tuple can be selected only once.
e Probability of picking any tuple = 1/N (all are equally likely).
e Example:
e If you have 1000 customer records and want a sample of 100, you randomly pick 100
different customers without repeating anyone.
2. Simple Random Sample With Replacement (SRSWR)
e We randomly select s tuples from dataset D , but after selecting a tuple, it is put back
into the dataset.
e So, the same tuple can be selected more than once.
e Every time, the probability of selecting any tuple stays 1/N.
e Example:
e If you randomly pick a customer record, you put it back into the pool after selection, so
the same customer might get picked multiple times in your sample of 100.
Method Replacement? Same tuple selected twice?
SRSWOR_ No No
SRSWR Yes Yes
3. Cluster Sampling
e Definition:
In cluster sampling, the entire data set (Say, dataset D) is divided into mutually disjoint
groups called clusters.

===== PAGE 11 =====
e Process:
Instead of selecting individual tuples (records) randomly, you randomly select entire
clusters.
e Example:
Suppose you have a database of 10,000 students from 100 schools.
Instead of sampling individual students, you randomly select, say, 10 schools (clusters)
and collect all students from these 10 schools.
e Key Points:
e Reduces effort because entire groups are selected at once.
e Objects within a cluster are usually similar.
e It is useful when it is difficult or expensive to create a complete list of individuals but
easy to list groups.
4. Stratified Sampling
e Definition:
In stratified sampling, the data set D is divided into strata (groups) based on some
important attribute.
Then, a simple random sample is drawn separately from each stratum.
e Process:
1. Divide the dataset into strata based on an attribute (like gender, age group, region, etc.).
2. Perform random sampling within each stratum.
e Example:
Suppose you have a database of employees where 60% are male and 40% are female.
To maintain this proportion, you separately sample males and females — ensuring that
your sample also reflects 60%-40% gender ratio.
e Key Points:
e Helps ensure that each subgroup is properly represented.
e Increases the accuracy and reliability of the sample.
e It is useful when the population is heterogeneous (diverse).

7. Real-world data tend to be incomplete, noisy and
inconsistent. What are the various approaches adopted to
clean the data?
Real-world data is often:

e Incomplete (missing values)

e Noisy (random errors)

e Inconsistent (contradictory values)

Data cleaning aims to fill missing values, remove noise, and fix inconsistencies.

1. Handling Missing Values
If some data (like customer income) is missing, we can:
(a) Ignore the Tuple

e Just skip the record if too much information is missing.

e Useful only if missing data is rare.
(b) Fill Missing Value Manually

e Human experts fill in missing info.

e Not practical for large datasets.
(c) Use a Global Constant

e Replace missing values with a label like "Unknown" .

e Can mislead analysis because "Unknown" looks like a real value.
(d) Use Attribute Mean

e Fill missing value with the average of that attribute.

e E.g., if average income = $56,000, use $56,000.

===== PAGE 15 =====
(e) Use Class-specific Mean
e If data is classified (like credit risk), fill missing value with mean for that class.
e E.g., average income for "high-risk" customers.
(f) Use Most Probable Value
e Use techniques like regression, Bayesian methods, or decision trees to predict missing
values.
2. Handling Noisy Data
Noise = random errors or variability in data.
Methods to smooth noisy data:
(a) Binning
e Sort data first.
e Divide into small groups ("bins").
e Then smooth:
e Bin mean: Replace each value with the bin’s average.
e Bin median: Replace with bin's median.
e Bin boundaries: Replace each value with nearest boundary (min or max).
(b) Clustering
e Group similar data points into clusters.
e Values far from any cluster are treated as outliers.
(c) Regression
e Fit a line (or curve) to the data.
e Predict missing or noisy values using this line.
e Linear regression (2 variables) or multiple regression (many variables).
3. Handling Inconsistent Data
Inconsistent data = data that conflicts or breaks rules.
How to handle:

===== PAGE 16 =====
e Manual correction: Cross-check with papers or trusted sources.
e Code standardization: Fix different code usages (e.g., USA vs US).
e Use knowledge rules: Apply known rules or constraints.
e Example: If state depends on ZIP code, mismatch indicates inconsistency.

11. Discuss the significance of data discretization in data
mining. List and explain any four data discretization
strategies.
Significance of Data Discretization in Data Mining
e Data Discretization is the process of converting a large set of continuous attribute values
into a smaller set of discrete intervals.
e It simplifies the data, making it easier to analyze, manage, and mine.
e Discretization reduces data size and helps in improving the performance of data mining
algorithms.
e It also helps in building concept hierarchies for further analysis.
e Example: Instead of handling hundreds of exact Age values, we can create intervals like
"Young", "Middle-aged", "Old".
1. Discretization by Binning
e Think of binning like putting data into small boxes.
e We divide the whole range into small parts (bins) — either all bins have the same size,
or each bin has the same number of values.
e Inside each bin, we can replace all the values with the average (mean) or middle value
(median) to make it even simpler.
e Example: If we have ages like 5, 7, 8, 9, 12, 13, 18 — we can group them into bins like:
e Bin 1: 0-10 — (5,7,8,9)

===== PAGE 23 =====
e Bin 2: 11-20 — (12,13,18)
2. Discretization by Histogram Analysis
e Imagine drawing a bar chart (histogram) of the data.
e Here also, data is grouped into bins (small ranges).
e Two ways to do it:
e Equi-width: Each bin has same width (like 0-10, 10-20, etc.).
e Equal-frequency: Each bin has same number of data points (like 5 values in each
bin).
e This helps see the data pattern and simplify it.
3. Discretization by Clustering, Decision Tree, and Correlation Analysis
a. Discretization by Clustering
e Clustering means grouping similar things together.
e Data points that are close or similar are put into one group (cluster).
e Each group acts like one interval (bin) for the data.
e It's like putting students into groups based on similar marks!
b. Discretization by Decision Tree
e Decision trees divide data step-by-step based on some conditions.
e It uses class labels (extra information) to decide where to cut the data.
e Example: In medical data, a tree can split patient records based on symptoms to predict a
disease.
e It tries to make groups as pure as possible (mostly same class inside a group).
c. Discretization by Correlation Analysis (ChiMerge)
e This method starts with each value as its own bin.
e Then, it merges bins that are similar based on a special test called Chi-square (x?) test.
e It keeps merging until the groups are good enough.
e It's like combining small groups into bigger ones when they look similar.

12. Illustrate PCA for dimensionality reduction with an
example.
What is PCA?
e PCAis a method to reduce the number of attributes (features) in your data.
e It finds new directions (principal components) along which the data varies the most.
e Then it replaces the original data with these few important directions.
e Goal: Keep the important patterns, remove extra dimensions.
PCA Step-by-Step
Suppose you have the following simple data of students:
Height (cm) Weight (kg)
150 50
160 55
165 58
170 62
175 65
You have 2 attributes: Height and Weight.
Now, let's apply PCA:
Step 1: Normalize the data
e Make sure height and weight are on the same scale.
Step 2: Find principal components
e PCA.will find new axes (directions) along which the data varies the most.
e Here, Height and Weight are strongly related (when height increases, weight increases).
e So PCA finds:
e 1st Principal Component (Y:): The direction where data varies the most (Height and
Weight together).

===== PAGE 25 =====
e 2nd Principal Component (Y2): A smaller variation direction (tiny differences not
captured by Y;).

Step 3: Sort components by importance

e Y, captures most of the pattern (Height + Weight relationship).

e Y2 Captures very little (random noise).
Step 4: Reduce dimensions

e We keep only Y: and discard Y2.

e Now instead of recording 2 things (height and weight), we just record one value along Y:.

e Dimension reduced from 2D to 1D

14, Explain Attribute Subset Selection
What is Attribute Subset Selection?
Attribute Subset Selection is a process used to select the most important attributes (or
features) from a dataset, and discard the irrelevant or redundant ones. The goal is to reduce the
complexity of the model by focusing on the most relevant attributes.
In simpler terms, instead of using every single piece of data in a dataset, we pick only the ones
that matter the most for making predictions. This helps in:
e Reducing complexity: We don't have to deal with too many variables.
e Improving performance: The model becomes faster and may perform better.

===== PAGE 31 =====
e Easier interpretation: It becomes easier to understand which attributes are most important.
How Does Attribute Subset Selection Work?
The process typically follows these steps
1. Start with all attributes: Initially, you have all the attributes (or features) from the dataset.
2. Evaluate the importance of each attribute: Look at how much each attribute contributes
to the prediction or classification task.
3. Remove unnecessary or redundant attributes: Eliminate the attributes that don’t help
much in making predictions.
4. Repeat: Keep selecting the best attributes until you have a small, efficient set.
There are different methods for selecting attributes:
e Stepwise Forward Selection
e Stepwise Backward Elimination
e Decision Tree Induction (which we'll cover in this example)
Example Dataset:
Let's say we have a small dataset of people with the following attributes:
Age’ Income Education Level Health Buy Product?

35 45,000 Master's Good Yes
50 70,000 = Bachelor's Poor No
28 40,000 = High School Good Yes
The task is to predict whether a person buys a product based on their Age, Income, Education
Level, and Health.
The target variable is Buy Product? (Yes/No).
Stepwise Forward Selection:

===== PAGE 32 =====
In Stepwise Forward Selection, we start with no attributes, and then add the most useful
attributes one at a time.
Step 1: Start with no attributes
At the beginning, we have no selected attributes.
Step 2: Evaluate the best attribute to start with
Now we evaluate the individual attributes (Age, Income, Education Level, Health) to see
which one has the most significant impact on predicting whether someone buys the product.
e We test each attribute independently and calculate how well each one predicts Buy
Product?. We might use a measure like accuracy or information gain to see how well
each attribute splits the data.
Let's say that Income gives the best result, so we choose Income as the first attribute.
Step 3: Add the best attribute to the model
Now, Income is our first chosen attribute, so the selected attributes are:
e Income
Step 4: Evaluate the next best attribute to add
Next, we evaluate which additional attribute works best with Income. We now have to evaluate
the combinations of Income and the remaining attributes (Age, Education Level, Health)
e Let’s say after testing combinations, Income + Age performs best.
Step 5: Add the best combination of attributes
Now, the selected attributes are:
e Income, Age
Step 6: Continue until no significant improvement
We keep adding attributes until we see no significant improvement in performance. For
example, adding Education Level and Health might not improve the prediction much, so we
stop.

15. Explain Data Integration and Data Transformation
In data mining, data integration and data transformation are two important preprocessing
steps that help prepare data for analysis
1. Data Integration
Data Integration refers to the process of merging data from multiple sources into a unified
dataset. This step is important because data often resides in various places: different
databases, files, or systems. Integrating this data ensures that all relevant information is
brought together for analysis.
Key Considerations in Data Integration:
1. Entity Identification Problem:

e When merging data from different sources, it’s important to ensure that equivalent
entities from various data stores are correctly matched. For example, a customer ID in
one database might be represented as customer_id, and in another database, it might
be cust_number . The system must recognize that these two fields refer to the same
entity.

e Solution: Use metadata (descriptive data about data) to help identify and match these
entities correctly.

2. Redundancy and Correlation Analysis:

e Redundancy occurs when the same information is stored in multiple places. For
instance, if two attributes (e.g., annual_income and yearly_salary ) provide the
same information, one can be removed to avoid redundancy.

e Correlation Analysis can be used to identify redundancies between attributes. For
instance, if two attributes show a high positive correlation (close to +1), it suggests that
one attribute might be redundant and can be removed.

3. Data Value Conflicts:

e Sometimes, data values from different sources may conflict. For example, one database

may store weights in kilograms, while another stores them in pounds. To handle this,

===== PAGE 34 =====
data needs to be standardized to a common format.
e Example: One database might record price in USD, while another records it in EUR.
These need to be normalized to the same currency for accurate analysis.

Example
Suppose we have customer data from two sources:

e Source 1: customer_id, name, age, address

e Source 2: cust_number, full_name, age, location
When integrating the data, we need to handle issues like matching customer_id with
cust_number and name with full_name . Additionally, we would handle potential
redundancies and conflicts in fields like address and location.
2. Data Transformation
Data Transformation involves converting data into a format suitable for mining. This could
mean scaling values, aggregating data, or simplifying complex data into more useful forms.
Key Data Transformation Methods:

1. Smoothing:

e Smoothing helps to remove noise from the data (i.e., inconsistencies or outliers).
Techniques like binning, regression, or clustering can be used to smooth the data
and make it more reliable for analysis.

e Example: If we have sales data with fluctuations due to occasional outliers, we might
smooth the data to get a clearer trend.

2. Aggregation:

e Aggregation involves summarizing data at a higher level. For example, instead of
tracking daily sales, you might aggregate the data to show monthly or yearly sales.

e Example: If the dataset contains daily sales data, aggregation could provide the total
sales for each month.

3. Generalization:

e Generalization replaces detailed data with higher-level concepts. For example, instead
of having a specific street address, we could generalize to a city or country.

e Example: Instead of tracking individual products, we might generalize the data to
product categories like "Electronics" or "Clothing."

===== PAGE 35 =====
4. Normalization:

e Normalization is the process of scaling data to fit within a specific range (like O to 1 or
-1 to 1). This is particularly important when attributes have different units or scales.
Common methods include min-max normalization and z-score normalization.

e Example: If we have data about income (ranging from $12,000 to $98,000) and age
(ranging from 18 to 70), normalization ensures that these attributes are on the same
scale for comparison.

5. Attribute Construction:

e In some cases, new attributes (or features) are created from existing ones to help
improve the analysis. This process is known as feature construction.

e Example: If we have data about height and weight, we might construct a new attribute
called BMI (Body Mass Index).
