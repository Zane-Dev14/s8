1. What are the requirements for a good clustering
algorithm?
1. Scalability
It should work well even if we have a large amount of data (both in terms of rows and
columns).
2. Ability to handle different shapes and sizes
It should be able to form clusters of different shapes (round, long, etc.) and sizes (big or
small).
3. High Quality Clusters
Objects in the same cluster should be similar to each other, and objects in different clusters
should be different from each other.

===== PAGE 5 =====
4. Noise and Outlier Handling
It should deal well with unusual data points (called outliers) and not get confused by them.
5. Minimal Requirements for Domain Knowledge
It should not require too much input like the number of clusters unless it's necessary.
6. Interpretability
The output (clusters) should be easy to understand and explain.
7. Deterministic Results
If we run it multiple times on the same data, it should give the same results (or almost the
same).
8. Minimal Parameters
It should not require setting too many parameters to work well.

A decision tree is like a flowchart that helps you make decisions or predictions by asking
yes/no questions step by step.
e It starts at the top with one main question (called the root node).

===== PAGE 6 =====
e Based on the answer, it follows a branch to the next question.
e This continues until it reaches a final answer (called a leaf node).
1. Overfitting
e The tree becomes too complex, capturing even small noise in the data.
e It works well on training data but gives poor results on new (test) data.
2. Instability
e Small changes in the data can cause big changes in the tree structure.
e This makes the model less reliable in real-world scenarios.
3. Bias Toward Features with Many Categories
e Ifa feature has many different values (like zip codes), the tree might focus on it too much.
e This can ignore other important features and reduce accuracy.
4. Hard to Prune
e After building the tree, we often need to remove unnecessary branches to avoid
overfitting.
e But deciding what to prune is not always easy and can affect performance.
5. Greedy Approachin SLIQ (and in decision trees in general), after we split once (for
example, by Temperature), we again apply the splitting logic inside each branch.
e First split: based on Temperature (example: Temperature < 72)
e Now, inside the new group (after splitting on Temperature),

Advantage of Gain Ratio over Information Gain:
Problem with Information Gain:
Information Gain likes attributes that give a clear split, even if that split is not useful.
For example:
e Suppose you have a column like Student ID.
e Each student has a unique ID, so if you split based on that, each group will have only one
student.
e This will give high Information Gain, because every group is "pure" (only one student = no
uncertainty).
e But it’s not helpful for prediction, because we can’t use ID to predict anything about other
students!
So, Information Gain gets tricked by attributes with many unique values.
How Gain Ratio Fixes This:
Gain Ratio looks at how many branches a split creates.
e If an attribute creates too many small groups, Gain Ratio penalizes it.
e It prefers splits that are useful and balanced, not just ones that look good by splitting too
much.
So, it helps us choose attributes that actually make meaningful decisions, not just those that
make perfect but useless splits.

e False Positive (FP): The model incorrectly predicted a negative as positive (e.g., Dog
is predicted as Cat).
e True Negative (TN): The model correctly predicted the negative class (e.g., Dog is
predicted as Dog).
e False Negative (FN): The model incorrectly predicted a positive as negative (e.g., Cat

7. Explain the concept of DBSCAN argorithm along with its

DBSCAN stands for:
Density-Based Spatial Clustering of Applications with Noise.
It's a clustering algorithm—which means it groups similar data points together.

===== PAGE 22 =====
Unlike other clustering methods like K-Means (which needs to know the number of clusters),
DBSCAN finds clusters based on how tightly the data points are packed (density).
Key Concepts
Imagine a map full of dots (data points). Some dots are close together (dense areas), and some
are spread out (Sparse areas). DBSCAN uses this idea of density to form clusters.
DBSCAN needs two settings:
1. Eps (€):
This is the distance to look around each point.
Like drawing a circle around a point — "Who’s close to me?"
2. MinPts (Minimum Points):
The minimum number of neighbors required to say:
“Yes, this is a dense region.”

1. Core Point:
A point that has at least MinPts within its €-radius.
— It's ina dense area.
2. Border Point:
Not enough neighbors to be core, but still near a core point.
— It’s on the edge of a cluster.
3. Noise Point (Outlier):
Neither a core nor near any core.
— It’s alone and doesn’t belong to any cluster.
How DBSCAN Works
1. Start with any point that hasn’t been visited.
2. Check how many neighbors it has within € distance.
e Ifithas MinPts or more — It's a core point — Create a new cluster.
e If fewer than MinPts = It may be a border or noise.
3. Add all neighbors of the core point to the same cluster.
4. For each new neighbor:
e If it’s a core point, check its neighbors and expand the cluster.
5. Repeat this process until all points are either:

e Labeled as noise.

11. Explain K-Means Clustering
What is K-Means?
e K-Means is a clustering algorithm.
e It groups data points into k clusters based on distance to cluster centers (called
centroids).
e The centroid is the mean (average) position of all points in the cluster — not necessarily
an actual data point.
Steps in K-Means:
1. Choose k (number of clusters).
2. Randonlly select k points as initial centroids.
3. Assign each data point to the nearest centroid.
4. Update centroids by calculating the mean of all points in each cluster.
5. Repeat steps 3 and 4 until centroids stop changing (or change very little).
{/ Final clusters are formed when the centroids become stable.

===== PAGE 39 =====
Example:
Suppose you have these points:
Points: (2, 6), (3, 4), (3, 8), (4, 7), (6, 2)
e Choose k = 2.
e Pick two random points as centroids, say (2,6) and (6,2).
e Assign points based on who they are closer to.
e Update the centroids (find mean of cluster points).
e Repeat until centroids stabilize.
12. Discuss PAM algorithm for clustering with an example.
e PAM is similar to k-means, but instead of using centroids (average points), it uses
medoids.
e Amedoid is an actual point from the dataset that best represents the cluster (has
minimum distance to others in the cluster).
e More robust to outliers than k-means because it doesn't create new imaginary points (like
k-means centroids do).
Key Terms:
Term Meaning
Medoid A real data point with the lowest total distance to other points in the cluster.
Cluster Group of points close to a medoid.
Dissimilarity Difference between points (using Manhattan or Euclidean distance).
PAM Algorithm Steps
1. Build Phase (Initialization)
e Choose k random points from the dataset as medoids.
e Assign each data point to the nearest medoid (based on distance).

2. Swap Phase (Optimization)
e Try swapping medoids with non-medoids.
e If swapping reduces the total distance (cost), accept the swap.
e Repeat until no swap reduces the cost.

13. Explain the working of SLIQ algorithm
What is SLIQ?
e SLIQ stands for Supervised Learning In Quest.
e Itis a decision tree algorithm made for handling large datasets.
e Special feature: It presorts numeric data before building the tree = making the tree
building much faster.
How SLIQ Works — Step by Step:
Step 1: Presort Numerical Attributes
e Before starting to build the tree, SLIQ sorts all numerical columns (like Temperature,
Humidity) only once.
e Example:
e Temperature: 64, 65, 68, 70, 72, 80, 83, 85
e Humidity: 65, 70, 78, 80, 85, 90, 95, 96
e Why?
> This saves time later when searching for the best split.
Step 2: Compute Splitting Criteria
e For categorical attributes (like Outlook, Wind):
e Use Information Gain or Gini Index to decide splits.
e For numerical attributes (like Temperature, Humidity):
e Quickly find the best split using the presorted list.

===== PAGE 42 =====
Step 3: Select the Best Splitting Attribute
e Among all attributes, choose the one that gives:
e Highest Information Gain (if using ID3 method)
e OR
e Lowest Gini Index (if using CART method)
e This attribute becomes the split at the node.
Step 4: Create a Class List
e Instead of copying full data at each node:
e SLIQ keeps a "class list": it only tracks class labels (like Yes, No).
Step 5: Recursively Grow the Decision Tree
e Keep repeating:
e Split nodes using the best attribute.
e Use the class list to guide.
e Stop when:
e All points in a node belong to the same class
e OR no good split can be found (then assign the majority class).
Step 6: Breadth-First Tree Building
e Unlike ID3 (which build depth-first),
> SLIQ builds level-by-level (breadth-first).
e > This makes it easier and faster for large datasets.
