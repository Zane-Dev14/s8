## Answer Writing Rule (BCT)

For every 14-mark answer:
1. Definition/context (2 to 3 lines)
2. Core architecture or formula block
3. Step-by-step workflow
4. Neat diagram (ASCII or boxed)
5. Advantages/limitations or security note
6. One-line conclusion

For coding questions:
1. Contract header + state variables
2. Constructor/initializer if needed
3. Required functions only (no extra noise)
4. Basic safety checks (require)
5. Short explanation of each function

This structure is the safest high-score format for CST428.
