# Module 2: Blockchain Architecture and Decentralization (Exam Sheet)

---

## Core Definition

Blockchain is a peer-to-peer distributed append-only ledger where state updates occur through consensus.

---

## Must-Know Terminologies

1. Transaction
2. Block
3. Previous hash
4. Nonce
5. Merkle root
6. Genesis block
7. Consensus

---

## Blockchain Working (7-Step Answer)

1. User creates and signs transaction.
2. Transaction broadcasts to network peers.
3. Nodes validate signatures and rules.
4. Valid transactions are grouped into block candidate.
5. Consensus approves next valid block.
6. Block links to previous hash and is appended.
7. Ledger state replicates across nodes.

## Centralized vs Decentralized

| Aspect | Centralized | Decentralized |
|---|---|---|
| Authority | Single | Shared protocol control |
| Failure model | Single point of failure | Fault tolerant replication |
| Trust basis | Institution | Protocol and cryptography |
| Censorship resistance | Lower | Higher |
